## 📊 搜索结果包含的信息

### 1. 标题 (Title)

**格式：**
```
{代币名} ({缩写}) Logo .SVG and .PNG Files Download - Cryptologo
```

**示例：**
- `Bitcoin (BTC) Logo .SVG and .PNG Files Download - Cryptologo`
- `Ethereum (ETH) Logo .SVG and .PNG Files Download - Cryptologo`
- `Fantom (FTM) Logo .SVG and .PNG Files Download - Cryptologo`

**特点：**
- ✅ 包含代币全称
- ✅ 包含代币缩写（如果有）
- ✅ 明确标注支持 SVG 和 PNG 格式
- ✅ 包含网站名称

### 2. URL (链接地址)

**格式：**
```
https://cryptologo.org/search/{搜索关键词}
```

**示例：**
- `https://cryptologo.org/search/btc-logo-svg`
- `https://cryptologo.org/search/btc-logo`
- `https://cryptologo.org/search/eth`
- `https://cryptologo.org/search/fantom-logo-svg`

**注意：** 用户点击后会自动重定向到实际的图标页面：
- `https://cryptologo.org/icon/bitcoin`
- `https://cryptologo.org/icon/ethereum`
- `https://cryptologo.org/icon/fantom`

### 3. 描述 (Description)

**格式：**
```
High quality raster (.PNG) and vector (.SVG) logo files for {代币信息}. {代币描述}. Official website: {官网}. Free download.
```

**示例：**
```
High quality raster (.PNG) and vector (.SVG) logo files for Bitcoin (BTC) - The first decentralized cryptocurrency. Official website: https://bitcoin.org. Free download.
```

**包含的关键信息：**
- ✅ **格式说明**：明确说明支持 PNG（位图）和 SVG（矢量）格式
- ✅ **代币信息**：代币全称和缩写
- ✅ **代币描述**：代币的简要介绍
- ✅ **官网链接**：代币的官方网站（如果有）
- ✅ **免费下载**：明确标注免费

### 4. 图片缩略图 (Image Thumbnail)

**来源：** 图标的 SVG 文件

**显示位置：** 搜索结果右侧或下方（取决于搜索引擎）

**示例：**
- Bitcoin 的橙色 Logo
- Ethereum 的蓝色 Logo
- Fantom 的蓝色 Logo

### 5. 结构化数据 (Rich Snippets)

搜索引擎可能会显示额外的结构化信息：

- **价格**：Free ($0)
- **可用性**：In Stock
- **格式**：SVG, PNG
- **类型**：Cryptocurrency Logo

## 🔄 搜索匹配逻辑

### 支持的搜索关键词格式

系统为每个图标自动生成多种搜索关键词：

#### 基于 Symbol（如 BTC）
- `btc-logo-svg`
- `btc-logo-png`
- `btc-logo`
- `btc-svg`
- `btc-png`
- `btc`

#### 基于 Name（如 Bitcoin）
- `bitcoin-logo-svg`
- `bitcoin-logo-png`
- `bitcoin-logo`
- `bitcoin-svg`
- `bitcoin-png`
- `bitcoin`

### 匹配优先级

1. **完全匹配 Symbol**（最高优先级，100分）
   - 搜索 `btc` → 匹配 Bitcoin

2. **完全匹配 Name**（90分）
   - 搜索 `bitcoin` → 匹配 Bitcoin

3. **完全匹配 ID**（80分）
   - 搜索 `bitcoin`（如果 ID 是 bitcoin）→ 匹配 Bitcoin

4. **部分匹配**（70-50分）
   - 搜索包含代币名称或缩写的关键词

5. **额外加分**
   - 查询包含 "logo" → +5分
   - 查询包含 "svg" 或 "png" → +5分
   - 根据 rank 调整（rank 越小越好）

## 🔍 如何验证搜索结果

### 1. 本地测试

构建项目后，访问搜索页面：

```bash
npm run build
npm run start
```

访问：
- `http://localhost:3000/search/btc-logo-svg`
- `http://localhost:3000/search/eth`
- `http://localhost:3000/search/fantom-logo`

### 2. 查看页面源码

在浏览器中查看页面源码，检查：
- `<title>` 标签
- `<meta name="description">` 标签
- OpenGraph 标签
- Twitter Card 标签
- JSON-LD 结构化数据

### 3. Google Search Console

部署后，在 Google Search Console 中：
1. 提交 sitemap
2. 查看索引状态
3. 使用 "URL 检查" 工具验证页面

### 4. 结构化数据测试

使用 Google 的 Rich Results Test：
- 输入页面 URL
- 查看结构化数据是否正确
- 检查是否有错误或警告

## 📝 注意事项

1. **搜索结果可能因搜索引擎而异**
   - Google、Bing、百度等可能显示略有不同
   - 描述长度可能被截断

2. **索引需要时间**
   - 新页面可能需要几天到几周才能被索引
   - 提交 sitemap 可以加快索引速度

3. **排名取决于多个因素**
   - 内容质量
   - 网站权威性
   - 用户点击率
   - 页面加载速度

4. **描述长度限制**
   - Google 通常显示 150-160 个字符
   - 超出部分会被截断
   - 重要信息应放在前面
