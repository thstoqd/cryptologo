# 如何配置 .env 文件

## 📝 步骤 1: 创建 .env 文件

在项目根目录（与 `package.json` 同级）创建 `.env` 文件：

```
cryptologo/
├── .env          ← 在这里创建
├── package.json
├── scripts/
└── ...
```

## ✏️ 步骤 2: 添加 API Key

在 `.env` 文件中添加以下内容（替换为你的实际 API Key）：

```env
COINMARKETCAP_API_KEY=你的API_KEY
```

**示例**：
```env
COINMARKETCAP_API_KEY=12345678-1234-1234-1234-123456789abc
```

## ✅ 步骤 3: 运行脚本

配置完成后，直接运行脚本即可：

```powershell
node scripts/update-ranks.js
```

不需要再手动设置环境变量了！

## 🔒 安全提示

- ✅ `.env` 文件已添加到 `.gitignore`，不会被提交到 GitHub
- ✅ 不要将 `.env` 文件分享给他人
- ✅ 如果 API Key 泄露，立即在 CoinMarketCap 中重新生成

## 📋 两种配置方式对比

### 方式 1: .env 文件（推荐本地开发）
- ✅ 方便，一次配置永久使用
- ✅ 不需要每次运行都设置环境变量
- ✅ 适合本地开发和测试

### 方式 2: 环境变量（临时）
```powershell
$env:COINMARKETCAP_API_KEY="你的API_KEY"
node scripts/update-ranks.js
```
- ⚠️ 每次关闭 PowerShell 后需要重新设置
- ✅ 适合临时测试

### 方式 3: GitHub Secrets（自动运行）
- ✅ 用于 GitHub Actions 自动运行
- ✅ 不需要本地配置
- ✅ 每天自动更新排名

## 🚀 快速开始

1. 复制 `.env.example` 为 `.env`：
   ```powershell
   copy .env.example .env
   ```

2. 编辑 `.env` 文件，填入你的 API Key

3. 运行脚本：
   ```powershell
   node scripts/update-ranks.js
   ```

完成！
